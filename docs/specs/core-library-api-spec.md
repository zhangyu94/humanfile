# Core Library API Specification

## Overview

This document defines the stable public API spec for the `humanfile` npm
package exported from `packages/core/src/index.ts`.

Exported API surface:
- `parse`
- `classify`
- `explain`
- `discoverRuleSets`
- `classifyAll`
- `ruleSetsForPath`
- `collectProtectedViolations`
- `evaluateAiHeuristic`
- `evaluateProtectedChangeSet`
- exported spec types (`ProtectionLevel`, `HumanRule*`, `Explain*`)
- exported policy types (`ChangedFileStat`, `ProtectedViolation`, `AiHeuristicResult`, `ProtectedPolicyEvaluation`)

## Shared Input Expectations

- Paths passed to API functions are expected to be repository-relative.
- Path separators are expected to use `/`.
- Rule sets are expected to be ordered from shallow to deep scope when passed directly to classifier functions.

## parse(content, directory?)

### Spec

- Parses `.human` text into a `HumanRuleSet`.
- Blank lines and `#` comment lines are ignored.
- Lines are trimmed before parsing.
- Plain patterns map to level `confirm`.
- `!`-prefixed patterns map to level `readonly`.
- A bare `!` line with no pattern is ignored.
- Returned rules preserve source order (last match wins behavior depends on this order).

## classify(filePath, ruleSets)

### Spec

- Returns one of: `free`, `confirm`, `readonly`.
- Within a single rule set, last matching rule wins.
- Across multiple rule sets, deeper (more specific directory) rule sets
  override shallower rule sets.
- Returns `free` when no rule matches.

## explain(filePath, ruleSets)

### Spec

- Returns final level and an explainability payload.
- `matched` is `true` only when at least one rule matched.
- `decisiveRule` is populated only when a match exists.
- `trace` preserves evaluation order and includes effective level after each rule evaluation.

## discoverRuleSets(repoRoot)

### Spec

- Walks the repository and discovers all `.human` files.
- Applies root `.gitignore` filtering semantics during discovery.
- Always excludes `.git`.
- Returns parsed rule sets sorted from shallowest to deepest directory scope.

## classifyAll(repoRoot)

### Spec

- Discovers rule sets and classifies all non-ignored files under `repoRoot`.
- Applies root `.gitignore` filtering semantics during file walk.
- Returns `Map<repoRelativePath, ProtectionLevel>`.
- `.human` files themselves are **not** included as keys in the returned map — they are parsed for rule sets but excluded from classification output.
- Classification semantics are the same as `classify`.

## ruleSetsForPath(filePath, ruleSets)

### Spec

- Filters rule sets to only those whose directory is a parent of (or equal to) the given file path.
- A root-scoped rule set (directory `""`) applies to all paths.
- Returned list preserves the input order (shallowest to deepest when sorted by `discoverRuleSets`).

## collectProtectedViolations(changedFiles, ruleSets)

### Spec

- Classifies each changed file against the provided rule sets.
- Returns only non-`free` entries.
- Returned records preserve `filename`, `additions`, `deletions`, and include resolved `level` (`confirm` or `readonly`).

## evaluateAiHeuristic(changedFiles, threshold, commitMessages?)

### Spec

- Computes `totalLinesChanged` as sum of additions plus deletions.
- Returns `likelyAiGenerated: true` when `totalLinesChanged >= threshold`.
- Otherwise returns `likelyAiGenerated: true` only when:
  - commit messages contain AI-signaling patterns, and
  - `totalLinesChanged > threshold / 2`.

### Commit message AI-signal patterns

The following case-insensitive patterns are checked against commit messages:

| Pattern                                                                 | Matches                   |
| ----------------------------------------------------------------------- | ------------------------- |
| `generated by (copilot\|cursor\|codex\|cline\|windsurf\|aider\|claude)` | Tool attribution          |
| `auto-generated`                                                        | Generic automation signal |
| `\bai[- ]assisted\b`                                                    | AI assistance label       |
| `co-authored-by:.*\b(copilot\|cursor\|codex\|cline\|windsurf\|aider)\b` | Git trailer attribution   |
| `\[bot\]$` (multiline)                                                  | Bot author suffix         |

## evaluateProtectedChangeSet(options)

### Spec

- Returns both policy violations and AI heuristic summary.
- `violations` are equivalent to `collectProtectedViolations(changedFiles, ruleSets)`.
- `likelyAiGenerated` and `totalLinesChanged` are derived from `evaluateAiHeuristic(...)`.

## Stability Guidance

Update this specification when any of the following changes:
- exported function names or exported types
- path input expectations
- precedence or matching semantics
- repository walk/discovery filtering behavior
- explain payload structure
